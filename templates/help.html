Introduction to Neural Networks

A neural network is a series of algorithms that endeavors to recognize underlying relationships in a set of data through a process that mimics the way the human brain operates.

In this application, you are training a neural network to recognize handwritten digits (0-9). The network takes the pixels of your drawing as input, passes them through hidden layers of neurons, and outputs a probability for each possible digit.

By tweaking the sandbox parameters, you can experiment with how different architectures and settings affect the network's learning speed and accuracy. Click on Sandbox Parameters in the sidebar to learn more about each setting.

Neural Network Sandbox Parameters

Welcome to the Neural Network Sandbox Help System. This guide will explain the various parameters you can tweak to train your digit-recognizing model.

Layer Sizes

A neural network is made of layers of "neurons". The Layer Sizes parameter defines how many neurons are in each layer.

  • 784: The input layer. Since the drawing pad is 28x28 pixels, there are 784 total pixels (28 * 28 = 784).
  • Hidden Layers: The middle numbers (e.g., 128, 64) are the hidden layers where the model learns complex patterns.
  • 10: The output layer. There are 10 possible digits (0-9).

Activations

Activation functions decide whether a neuron should "fire" or not, allowing the network to learn non-linear patterns.

  • relu: Rectified Linear Unit. The standard default. It outputs the input directly if it is positive, otherwise, it outputs zero.
  • sigmoid: Squeezes values between 0 and 1. Good for probabilities but can suffer from vanishing gradients in deep networks.
  • tanh: Squeezes values between -1 and 1.
  • linear: No activation. Often used on the final output layer before a loss function is applied.

Loss & Optimizers

The Loss Function (e.g., cross_entropy) calculates how "wrong" the model's predictions are. The Optimizer (e.g., adam, sgd) is the algorithm that adjusts the model's weights to minimize that loss.

Learning Rate (LR)

The learning rate determines how big of a step the optimizer takes when adjusting weights. If it's too high, the model might overshoot the optimal solution. If it's too low, training will be very slow.

Epochs & Batch Size

An Epoch is one complete pass through the entire training dataset. The Batch Size is how many images the model looks at before updating its weights.
